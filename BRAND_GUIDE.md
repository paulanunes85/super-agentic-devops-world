## 1. Brand Identity

### Brand Name

**Super Agentic DevOps World**

### Subtitle / Tagline

**Made for Sofia**

### Secondary Taglines (for different contexts)

| Context | Tagline |
|---------|---------|
| Full scope | "8 Worlds. 69 Levels. Infinite Power-Ups." |
| Onboarding | "Press START to learn DevOps & AI" |

### Brand Story

Sofia is the daughter of **Paula Silva** (Microsoft Latam Software GBB). When Sofia asked her mother to teach her about software development, Paula realized that traditional tutorials, documentation, and courses weren't designed for someone starting from absolute zero. The jargon was intimidating, the concepts felt abstract, and the learning curve felt like a wall.

So Paula decided to explain everything through something Sofia already loved: **video games**. Specifically, **Super Mario World** — a game where the rules are simple, the progression is clear, and anyone can play.

What started as a mother teaching her daughter turned into something much bigger. If Paula could make Sofia — someone with zero tech background — understand Git, CI/CD, AI Agents, RAG, and Kubernetes through Mario analogies, then **ANYONE** could learn these concepts the same way.

**"Made for Sofia"** is more than a subtitle. It's a promise: this content was built with the same care, patience, and love that a parent puts into teaching their child. No jargon without explanation. No concept without an analogy. No level too hard without the right power-ups.

Every developer starts at World 1-1. Sofia did. And now, so can you.

### Brand Promise

We turn complex DevOps and AI Agent concepts into something **ANY** person can understand, using the universal language of video games.

### Brand Pillars

| Pillar | Description |
|--------|-------------|
| **Accessibility** | No prior knowledge required. World 1-1 starts from absolute zero. |
| **Analogy-first** | Every technical concept is introduced through a Super Mario World analogy before the real-world explanation. |
| **Progressive difficulty** | Content follows the game structure — each World builds on the last. |
| **Joy in learning** | Games are fun. Learning should be too. Every milestone is celebrated. |
| **Real-world readiness** | By World 8, learners can build and deploy AI-powered DevOps pipelines. |

1. **Never use more than 3 colors** on a single page or slide (excluding backgrounds).

3. **Coin Gold** is reserved for achievements, celebrations, and accent details.
4. **World Colors** should only be used within their corresponding World's content.
5. **Always maintain sufficient contrast** — minimum WCAG AA compliance (4.5:1 for text).

## 3. Typography

### Recommended Fonts

| Usage | Font | Fallback | Why |
|-------|------|----------|-----|
| **Headings** | **Press Start 2P** (Google Fonts) | `"Courier New", monospace` | Pixel font that screams retro gaming |
| **Body Text** | **Inter** or **Segoe UI** | `system-ui, sans-serif` | Clean, readable, professional |
| **Code** | **JetBrains Mono** or **Cascadia Code** | `"Courier New", monospace` | Developer-friendly monospace |
| **Accent/Callouts** | **Press Start 2P** at small size | `monospace` | For "POWER-UP UNLOCKED!" moments |

### Font Hierarchy

| Element | Font | Size | Color | Weight |
|---------|------|------|-------|--------|
| H1 | Press Start 2P | 24px | Mario Red `#E52521` | Normal |
| H2 | Press Start 2P | 18px | Sky Blue `#049CD8` | Normal |
| H3 | Inter | 16px | Dark `#1A1A2E` | Bold |
| Body | Inter | 14px | Dark `#1A1A2E` | Regular |
| Code | JetBrains Mono | 13px | Light on dark | Regular |
| Callout | Press Start 2P | 12px | Coin Gold `#FBD000` | Normal |

### Typography Rules

1. **Press Start 2P** should only be used for headings, callouts, and short phrases. Never for body text — pixel fonts are hard to read at length.
2. **Line height** for body text: 1.6 minimum. Readability is a Power-Up.
3. **Code blocks** always use the dark background (`#2D2D2D`) with light text.
4. **Never mix more than 2 fonts** on a single page (heading font + body font).

### Loading the Fonts

```html
<!-- Google Fonts -->
<link href="https://fonts.googleapis.com/css2?family=Press+Start+2P&family=Inter:wght@400;600;700&family=JetBrains+Mono&display=swap" rel="stylesheet">
```

```css
/* CSS Variables */
:root {
  --font-heading: 'Press Start 2P', 'Courier New', monospace;
  --font-body: 'Inter', 'Segoe UI', system-ui, sans-serif;
  --font-code: 'JetBrains Mono', 'Cascadia Code', 'Courier New', monospace;
}
```

## 6. Tone of Voice

### Core Principle

**If a 12-year-old cannot understand it, simplify it.**

The brand voice is warm, encouraging, playful, and technically accurate. Think of it as a patient friend who happens to be a senior engineer and also loves video games.

### Do's

- Use simple, clear language at all times
- Use Mario references naturally (not forced)
- Be encouraging — "You got this!" not "This is hard but..."
- Use humor — games are FUN, learning should be too
- Use progressive difficulty — never throw World 8 concepts at a World 1 player
- Celebrate progress — every small win is a coin collected
- Explain jargon through analogy FIRST, then give the real definition
- Use "we" and "you" — it is a shared journey

### Don'ts

- Never be condescending — "This is easy" makes people feel bad if they struggle
- Never use jargon without explaining it first through a Mario analogy
- Never assume prior knowledge — World 1-1 assumes ZERO experience
- Never skip the analogy — every concept gets its Mario equivalent
- Don't overuse exclamation marks — enthusiasm yes, screaming no
- Never gatekeep — phrases like "you should already know this" are banned
- Never use passive voice when active voice is clearer

### Voice Examples

| Instead of... | Say... |
|--------------|--------|
| "Configure the CI/CD pipeline" | "Set up your Lakitu workers to automate the boring stuff" |
| "This is a complex topic" | "This is a World 5 concept — you've come far!" |
| "Error in your code" | "A Goomba appeared! Let's squash it" |
| "You completed the module" | "POWER-UP UNLOCKED! You now know Git!" |
| "Prerequisites required" | "Before entering this level, make sure you have..." |
| "API authentication failed" | "The castle gate is locked — we need the right key" |
| "Deployment successful" | "Your code made it to the flagpole! Level complete." |
| "This section covers containers" | "Time to learn about Docker — think of it as Mario's pipes" |

### Writing Checklist (for every piece of content)

- [ ] Does the intro have a Mario analogy?
- [ ] Is jargon explained before it is used?
- [ ] Would a complete beginner understand this?
- [ ] Is there at least one celebration moment (Power-Up, Badge, etc.)?
- [ ] Does the tone feel like a friend, not a textbook?

## 10. Presentation Template

### Slide Structure for Workshops

| Slide | Content | Design Notes |
|-------|---------|-------------|
| **Slide 1** | Title — "Super Agentic DevOps World" logo, "Made for Sofia", speaker info | Centered, Mario Red title, parchment background |
| **Slide 2** | "Press START" — What we'll cover today (World selection screen) | Grid layout mimicking game world map |
| **Slides 3-N** | Content — Left side: Mario analogy visual / Right side: Technical content | Split layout, 50/50 or 60/40 |
| **Slide N+1** | "POWER-UP UNLOCKED!" — Summary of what was learned | Coin Gold accents, celebration feel |
| **Slide Final** | "Continue?" — Next steps, resources, QR code to guide | Call to action, social links |

### Slide Color Scheme

| Element | Light Mode | Dark Mode |
|---------|-----------|-----------|
| Background | `#F5F0E1` (parchment) | `#1A1A2E` (night sky) |
| Headers | `#E52521` (Mario Red) | `#E52521` (Mario Red) |
| Accents | `#FBD000` (Coin Gold) | `#FBD000` (Coin Gold) |
| Code blocks | `#2D2D2D` background | `#2D2D2D` background |
| Body text | `#1A1A2E` | `#F5F0E1` |

### Slide Design Rules

1. **One concept per slide** — if you need to scroll, split it.
2. **Every technical slide has a Mario visual** — never just text.
3. **Use the pixel font (Press Start 2P) only for slide titles** — body text uses Inter/Segoe UI.
4. **Maximum 6 bullet points per slide** — less is more.
5. **Always include the World number** in the slide footer so attendees know where they are.
6. **Transitions**: Use simple fades. No flashy animations — the content is the star.
